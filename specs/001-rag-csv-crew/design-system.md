# Design System Specification: HTML Response Formatting

**Feature**: `001-rag-csv-crew`
**Created**: 2026-02-02
**Purpose**: Define measurable design standards for HTML-formatted query responses per FR-008

## Overview

This document specifies the design system for HTML-formatted responses generated by the RAG system. All responses MUST follow these standards to ensure readability, accessibility, and consistent user experience.

## Typography

### Font Family

```css
/* Primary font stack (sans-serif) */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;

/* Monospace font stack (for code, data values) */
font-family: "SF Mono", Monaco, "Cascadia Code", "Roboto Mono", Consolas, monospace;
```

**Rationale**: System font stack ensures native look-and-feel, fast loading (no web fonts), and OS-level rendering optimizations.

### Font Sizes

| Element | Size | Line Height | Weight | Use Case |
|---------|------|-------------|--------|----------|
| h1 (Main Answer) | 24px (1.5rem) | 1.25 (30px) | 600 (Semi-Bold) | Primary answer summary |
| h2 (Sections) | 20px (1.25rem) | 1.3 (26px) | 600 (Semi-Bold) | Answer sections, categories |
| h3 (Subsections) | 18px (1.125rem) | 1.35 (24px) | 600 (Semi-Bold) | Detailed breakdowns |
| Body Text (p, li) | 16px (1rem) | 1.5 (24px) | 400 (Normal) | Explanatory text, descriptions |
| Small Text (captions) | 14px (0.875rem) | 1.4 (20px) | 400 (Normal) | Metadata, confidence scores |
| Code/Data Values | 14px (0.875rem) | 1.5 (21px) | 400 (Normal) | SQL queries, numeric data |

**Base Size**: 16px (1rem) - browser default for accessibility
**Scale**: Major third ratio (1.25×) for headings

### Text Color

**Accessibility Target**: WCAG 2.1 Level AA minimum (4.5:1 for normal text, 3:1 for large text)

## Semantic HTML Structure

### Required Elements

All responses MUST use semantic HTML5 elements:

```html
<article>
  <header>
    <h1>Main Answer Summary</h1>
    <aside class="metadata">
      <span class="confidence">Confidence: 92%</span>
      <span class="timestamp">2026-02-02 14:30:00</span>
    </aside>
  </header>

  <section>
    <h2>Section Title</h2>
    <p>Explanatory paragraph text.</p>

    <table>
      <thead>
        <tr>
          <th>Column Header</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Data value</td>
        </tr>
      </tbody>
    </table>
  </section>

  <footer>
    <p class="note">Additional context or disclaimers.</p>
  </footer>
</article>
```

### Element Usage Guidelines

| Element | Purpose | When to Use |
|---------|---------|-------------|
| `<article>` | Container for entire response | Always (root element) |
| `<header>` | Response metadata (confidence, timestamp) | Always |
| `<h1>` | Main answer summary | Always (one per response) |
| `<h2>` | Major sections (e.g., "Top Results", "Analysis") | When answer has multiple sections |
| `<h3>` | Subsections within `<h2>` sections | For detailed breakdowns |
| `<section>` | Thematic grouping of content | When answer has distinct logical sections |
| `<p>` | Explanatory paragraphs | For prose text, explanations |
| `<ul>` | Unordered lists | For non-sequential items (categories, tags) |
| `<ol>` | Ordered lists | For ranked results, sequential steps |
| `<table>` | Tabular data | When data has rows and columns |
| `<aside>` | Tangential information (warnings, notes) | For confidence scores, metadata |
| `<footer>` | Response disclaimers, context | When additional context needed |
| `<code>` | Inline data values, SQL fragments | For column names, numeric values |
| `<pre><code>` | Full SQL queries | When showing generated SQL |

### Prohibited Elements

DO NOT use:
- `<div>` and `<span>` (except with semantic class names for styling)
- `<b>` and `<i>` (use `<strong>` and `<em>` for semantic meaning)
- Inline styles (use class names instead)
- `<font>`, `<center>`, `<big>`, `<small>` (deprecated)

## Responsive Behavior

### Breakpoints

```css
/* Mobile-first responsive design */
--mobile: 320px;       /* Small phones */

--desktop: 1024px;     /* Desktop screens */

```

### Responsive Adjustments

| Element | Mobile (<768px) | Tablet (768-1023px) | Desktop (≥1024px) |
|---------|-----------------|---------------------|-------------------|
| h1 font-size | 20px (1.25rem) | 22px (1.375rem) | 24px (1.5rem) |
| Body font-size | 16px (1rem) | 16px (1rem) | 16px (1rem) |
| Padding (article) | 1rem | 1.5rem | 2rem |
| Table overflow | scroll (horizontal) | scroll (if needed) | no scroll |
| Column layout | 1 column | 2 columns | 2-3 columns |

**Touch Targets**: All interactive elements (links, buttons) MUST be minimum 44×44px for mobile accessibility.

## Accessibility (WCAG 2.1 AA)

### Compliance Checklist

- ✅ **Contrast Ratios**: Minimum 4.5:1 for normal text, 3:1 for large text (18px+ or 14px+ bold)
- ✅ **Semantic HTML**: Use semantic elements (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`)
- ✅ **Headings Hierarchy**: Logical heading structure (h1 → h2 → h3, no skipping levels)
- ✅ **Alt Text**: All images MUST have descriptive alt attributes (if images used)
- ✅ **Keyboard Navigation**: All interactive elements focusable and operable via keyboard
- ✅ **Focus Indicators**: Visible focus outlines (2px solid #0066cc, 4px offset)
- ✅ **Table Headers**: Use `<th scope="col">` for column headers, `<th scope="row">` for row headers
- ✅ **List Structure**: Use proper list markup (`<ul>`, `<ol>`, `<li>`)
- ✅ **Language Attribute**: Specify `lang="en"` on root element
- ✅ **Resize Text**: Content readable and functional at 200% text size

### Screen Reader Considerations

- **ARIA Labels**: Use `aria-label` for confidence indicators (e.g., `aria-label="Confidence level: 85 percent, high confidence"`)
- **ARIA Live Regions**: Not required (responses are static, not dynamically updating)
- **Skip Links**: Not required (responses are standalone content, not full pages)

## Implementation Requirements

### CSS Class Naming Convention

Use BEM (Block Element Modifier) naming:

```css
/* Block */
.response { }

/* Element */
.response__header { }
.response__section { }
.response__confidence { }

/* Modifier */
.response__confidence--high { }
.response__confidence--low { }
```

### HTML Generation Template

```python
def generate_html_response(
    summary: str,
    sections: list[dict[str, Any]],
    confidence: float,
    timestamp: str
) -> str:
    """Generate HTML-formatted response following design system."""
    html: str = f"""
    <article class="response">
      <header class="response__header">
        <h1>{escape_html(summary)}</h1>
        <aside class="response__metadata">
          <span class="confidence-indicator confidence-indicator--{get_confidence_level(confidence)}">
            Confidence: {confidence:.0f}%
          </span>
          <span class="timestamp">{timestamp}</span>
        </aside>
      </header>

      {generate_sections(sections)}

      <footer class="response__footer">
        <p class="note">Generated by Hybrid Search RAG System</p>
      </footer>
    </article>
    """
    return html
```

## Validation

### Automated Checks

All generated HTML responses MUST pass:

1. **HTML Validation**: W3C HTML5 validator (no errors)
2. **Accessibility Validation**: axe-core automated testing (no violations)
3. **Contrast Validation**: WebAIM Contrast Checker (WCAG 2.1 AA minimum)
4. **Responsive Validation**: Viewable on mobile (320px) through desktop (1920px)

### Manual Checks

- Screen reader testing (NVDA/JAWS on Windows, VoiceOver on macOS)
- Keyboard navigation (Tab, Shift+Tab, Enter, Space)
- Browser compatibility (Chrome, Firefox, Safari, Edge latest versions)
